PrepPal backend integration is ready for demo/testing.

Active backend configuration (source of truth: lib/core/network/api_constants.dart)

Main API base URL
- https://preppal-backend-px1d.onrender.com

ML forecast base URL
- https://preppal-2mb4.onrender.com
- Predict endpoint: /api/predict

Current auth endpoints
- Login: /api/v1/auth/login
- Signup: /api/v1/auth/signup
- Forgot password: /api/v1/auth/forgot-password
- Reset password: /api/v1/auth/reset-password

Current business endpoints
- Create: /api/v1/business/create
- Get all (for signed-in user): /api/v1/business
- Update: /api/v1/business/update/{id}

Current inventory endpoints
- Create: /api/v1/inventory/create
- Get by business: /api/v1/inventory/business/{businessId}
- Update/Delete: /api/v1/inventory/{id}

Current daily sales endpoints
- Get all: /api/v1/daily-sales/sales
- Create: /api/v1/daily-sales/create
- Get by business: /api/v1/daily-sales/business/{businessId}

Current alerts endpoints
- Get all: /api/v1/alerts
- Get by business: /api/v1/alerts/business/{businessId}

Network behavior in app
- Auth token handled automatically with Bearer header.
- Retry enabled for transient network issues (up to 2 retries).
- Timeout budget is connection + receive time from ApiConstants.

Forecast integration notes
- Demand forecast UI uses the ML /api/predict endpoint.
- Partial ML failures are handled gracefully so the screen still renders.

Recommended demo flow
1. Login or sign up.
2. Complete business details.
3. Add inventory.
4. Open Forecast tab and refresh.
5. Open Dashboard/Alerts to show downstream impact.
